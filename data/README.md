# Substrate Data Contract

JSONL is the canonical store. Each line is one complete JSON object. Files are append-only unless a human explicitly approves a cleanup migration.

Before any automated run writes substrate data, it must acquire the repo-level write lock:

```bash
npm run substrate:lock acquire
```

Release it at the end of the run:

```bash
npm run substrate:lock release
```

If another instance holds the lock, stop the run instead of writing. Stale locks older than six hours are removed automatically by the lock script.

Before committing, run:

```bash
npm run substrate:check
```

This fails on duplicate canonical IDs and warns on likely duplicate content keys.

## Files

- `sources.jsonl`: official source batches and rights notes.
- `items.jsonl`: normalized assessment items or safe item metadata.
- `misconceptions.jsonl`: misconception hypotheses grounded in source evidence.
- `concept-packs.jsonl`: tutor-ready diagnostic/intervention bundles.
- `runs.jsonl`: one record per nightly run.
- `review-index.csv`: flat mirror for spreadsheet review. It is not canonical.

## Required Record Shape

IDs must be deterministic, not generated from the current time. If the same source/item/misconception is encountered by two independent runs, both runs should produce the same ID so duplicates are detectable.

Recommended ID rules:

- `source_id`: agency + assessment + year + grade + subject + material family.
- `item_id`: `source_id` + official item number/id when available; otherwise `source_id` + stable content/metadata hash.
- `misconception_id`: subject + grade band + normalized concept + normalized error pattern.
- `concept_pack_id`: subject + grade band + normalized concept + version only when the instructional model changes.
- `run_id`: date + source batch id + short hash of the run scope.

### Source

```json
{
  "source_id": "tea-staar-2025-g4-math",
  "agency": "Texas Education Agency",
  "url": "https://...",
  "retrieved_at": "2026-06-06",
  "year": 2025,
  "grade": 4,
  "subject": "math",
  "materials": ["test_form", "answer_key", "item_rationale", "student_expectations"],
  "rights_note": "Official released assessment material. Do not republish full passages or complete forms without explicit rights review."
}
```

### Item

```json
{
  "item_id": "tea-staar-2025-g4-math-item-12",
  "source_id": "tea-staar-2025-g4-math",
  "grade": 4,
  "subject": "math",
  "standard_refs": ["4.NF.A.1"],
  "concept": "equivalent fractions",
  "item_type": "multiple_choice",
  "answer_metadata": {"correct_option": "C"},
  "safe_summary": "Equivalent-fraction item involving multiplicative scaling.",
  "rights_note": "Metadata and short summary only; full item text not stored."
}
```

### Misconception

```json
{
  "misconception_id": "math-g4-equivalent-fractions-additive-scaling",
  "evidence_refs": ["tea-staar-2025-g4-math-item-12"],
  "claim_status": "inferred",
  "confidence": "medium",
  "grade_band": "3-5",
  "subject": "math",
  "concept": "equivalent fractions",
  "student_error": "Treats numerator and denominator as values to change additively rather than as a ratio scaled multiplicatively.",
  "diagnostic_signal": "Chooses an option generated by adding the same amount to numerator and denominator.",
  "intervention": "Use area model and n/n identity reasoning before returning to symbolic form.",
  "exit_check": "Student identifies an equivalent fraction generated by multiplying both terms by the same factor.",
  "tutor_action": "ask_visual_comparison_then_retest"
}
```

```json
{

  "evidence_refs": ["tea-staar-2025-g4-math-item-12"]
}
```

## Sheet Mirror

If a Google Sheet is created later, mirror only flat review fields:

`run_date, source_id, grade, subject, concept, misconception_id, confidence, claim_status, evidence_count, pitch_ready, review_status`

Do not make the Sheet the source of truth. It cannot safely preserve the nested evidence graph.
